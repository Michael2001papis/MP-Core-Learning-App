סריקת פרויקט לפני עבודה — MP-Core-Learning-App
תאריך: 2026-03-18

סיכום קצר
---------
זהו פרויקט Frontend סטטי (Client-Side בלבד) בעברית RTL, מבוסס Vite, לא SPA אלא Multi-page.
הקוד הוא HTML/CSS/Vanilla JS. יש בדיקות Unit עם Vitest ובדיקות E2E עם Playwright.

מידע סביבתי
-----------
- מערכת הפעלה (לפי סביבת העבודה): Windows 10
- נתיב פרויקט: d:\WEBS\MP-Core-Learning-App
- מצב: לא מזוהה כ-repo של Git בסביבת העבודה הנוכחית

מבנה תיקיות וקבצים בשורש
------------------------
תיקיות:
- assets/        נכסים (למשל favicon)
- css/           עיצוב גלובלי
- dist/          פלט build סטטי (קיים כרגע)
- e2e/           בדיקות Playwright
- js/            מודולים משותפים
- node_modules/  תלויות (קיים כרגע)
- pages/         דפים (Multi-page)

קבצים בולטים:
- index.html
- package.json / package-lock.json
- vite.config.js / vitest.config.js / playwright.config.js
- build-static.js
- manifest.json (PWA)
- sw.js (Service Worker)
- sitemap.xml
- vercel.json
- README.md / LICENSE / CREDITS

טכנולוגיות ותלויות (מתוך package.json)
---------------------------------------
- Tooling: Vite
- Unit tests: Vitest (+ @vitest/ui), jsdom
- E2E: @playwright/test

Scripts:
- npm run dev        הפעלת שרת פיתוח (Vite)
- npm run build      build סטטי באמצעות build-static.js (העתקה ל-dist)
- npm run build:vite build מלא דרך vite build (קיים אך לא הסקריפט הראשי לפריסה)
- npm run preview    Vite preview
- npm test           Vitest
- npm run test:ui    Vitest UI
- npm run test:e2e   Playwright test

הרצה מקומית (מומלץ)
-------------------
1) התקנת תלויות:
   npm install

2) פיתוח:
   npm run dev

3) בדיקות:
   npm test -- --run
   npm run test:e2e

4) פריוויו לאחר build:
   npm run build
   npm run preview

הערת Build חשובה
----------------
סקריפט ה-build הראשי הוא build-static.js שמבצע העתקה סטטית של הקבצים ל-dist ללא עיבוד Vite.
הוא מעתיק: index.html, sitemap.xml, manifest.json, sw.js, LICENSE, CREDITS, והתיקיות js/css/assets/pages.

הגדרות Multi-page (vite.config.js)
----------------------------------
Vite מוגדר עם מספר entry points תחת rollupOptions.input:
- index.html (main)
- pages/home/index.html
- pages/about/index.html
- pages/contact/index.html
- pages/privacy/index.html
- pages/terms/index.html
- pages/404/index.html
- pages/games/tic-tac-toe/index.html
- pages/games/snake/index.html
- pages/login/index.html

בדיקות (Vitest + Playwright)
----------------------------
Vitest (vitest.config.js):
- environment: jsdom
- include: **/*.test.js ו-**/*.spec.js
- exclude: node_modules, dist, pages/**
  המשמעות: בדיקות היחידה אינן מכסות את סקריפטי הדפים תחת pages/ כרגע.

Playwright (playwright.config.js):
- testDir: ./e2e
- baseURL: http://localhost:5173
- webServer: מריץ npm run dev ומחכה ל-5173
- projects: chromium + firefox

PWA ו-Offline
-------------
manifest.json:
- start_url: /pages/home/index.html
- display: standalone
- dir: rtl, lang: he
- icons: /assets/favicon.svg (כולל maskable)

sw.js:
- CACHE_NAME: gamehub-v3
- Precache: כולל דפי home/login/about/contact/privacy/terms/404 וגם המשחקים
- אסטרטגיה: Network-First עם fallback ל-cache כאשר אין אינטרנט
- שמירת responses ל-cache עבור ניווט וקבצי js/css/svg/json (ובתנאים מסוימים)

ממצאים חשובים / נקודות סיכון לפני שינויי קוד
---------------------------------------------
1) פער Naming ב-Analytics:
   - בקובץ js/analytics.js מוגדר window.GameHubAnalytics
   - במשחק Snake (pages/games/snake/script.js) יש ציפייה ל-window.ANALYTICS.track
   - המשמעות: אם analytics מופעל/נצרך במשחקים, ייתכן שאין מעקב בפועל או שיש חוסר עקביות.

2) קבצים “לא בשימוש” לפי README קיימים בפועל:
   - pages/login/js/email-config.js
   - pages/login/vendor/email.min.js
   קיימים גם עותקים תחת dist/ (כחלק מהעתקה סטטית ב-build).

3) index.html בשורש משתמש בשתי אסטרטגיות הפניה:
   - meta refresh
   - location.replace
   (לא שגיאה, אבל כפילות שיכולה לבלבל/ליצור התנהגות כפולה במקרים מסוימים.)

4) dist/ ו-node_modules/ קיימים כבר:
   - dist/ הוא תוצר build וניתן למחיקה/בנייה מחדש
   - node_modules/ מצביע על כך שכבר הותקנו תלויות (עדיין מומלץ לוודא npm install מצליח)
