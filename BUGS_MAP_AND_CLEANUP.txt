מיפוי באגים קלים + ניקוי מומלץ — MP-Core-Learning-App
תאריך: 2026-03-18

תקציר
------
המסמך מרכז “באגים קלים” (שגיאות קונפיג/פערי API/נקודות שעלולות לייצר תקלות) + הצעות ניקוי בטוחות.
בנוסף רשום מה תוקן בפועל במהלך העבודה ומה נשאר כהמלצה.

מה תוקן (בוצע בפועל)
--------------------
1) Analytics — תאימות ל-API שונה במשחק Snake
   - חומרה: נמוכה-בינונית (פיצ'ר/טלמטריה שלא פועלת, בלי קריסה)
   - מיקום:
     - שימוש: pages/games/snake/script.js מצפה ל-window.ANALYTICS.track(...)
     - הגדרה: js/analytics.js מגדיר window.GameHubAnalytics בלבד
   - תיקון:
     - הוסף alias תואם לאחור ב-js/analytics.js:
       אם window.ANALYTICS לא קיים → מוגדר { track(name, props) { event(name, props) } }
   - סטטוס: תוקן ✅

2) Vitest מריץ בטעות קבצי E2E (Playwright) כ-Unit tests
   - חומרה: בינונית (שובר CI/הרצה מקומית של בדיקות unit)
   - סימפטום:
     - npm test נכשל בגלל e2e/home.spec.js עם שגיאת Playwright (“test.describe() …”)
   - שורש בעיה:
     - vitest.config.js כולל **/*.spec.js אבל לא מוציא e2e/**
   - תיקון:
     - vitest.config.js: נוספה החרגה e2e/**
   - סטטוס: תוקן ✅

3) npm test נתקע ב-watch mode (DX)
   - חומרה: נמוכה (לא באג מוצר, אבל מעצבן וגורם “תהליך שלא נסגר”)
   - שורש בעיה:
     - package.json: "test": "vitest" (ברירת מחדל DEV/watcher)
   - תיקון:
     - package.json: "test": "vitest run"
     - נוסף: "test:watch": "vitest"
   - סטטוס: תוקן ✅

4) Playwright config — שגיאת תחביר שחוסמת E2E
   - חומרה: גבוהה (מונע הרצת E2E בכלל)
   - מיקום: playwright.config.js
   - שורש בעיה:
     - חסר סוגר/סגירה במערך projects (Firefox project לא נסגר)
   - תיקון:
     - סגירת האובייקט + סגירת המערך projects
   - סטטוס: תוקן ✅

ממצאים/המלצות ניקוי (לא בוצע מחיקה בפועל)
-----------------------------------------
1) EmailJS — קבצים שמסומנים כ”לא בשימוש” קיימים בפועל
   - חומרה: נמוכה (רעש/משקל/תחזוקה)
   - מיקום:
     - pages/login/js/email-config.js
     - pages/login/vendor/email.min.js
   - מצב:
     - לפי התיעוד בקובץ עצמו: “לא בשימוש”
     - לא נראית טעינה שלהם ב-pages/login/index.html (כלומר לא בשימוש בפועל)
   - ניקוי מומלץ:
     - אם אין תכנון להפעיל איפוס סיסמה במייל: להסיר את 2 הקבצים האלה (וגם את העותקים תחת dist/ בבנייה הבאה)
     - אם כן רוצים: לחבר UI/לוגיקה ולהוציא מפתחות ל-.env (לא לשמור בקוד)

2) הפניה כפולה ב-index.html (meta refresh + location.replace)
   - חומרה: נמוכה
   - מיקום: index.html
   - ניקוי מומלץ:
     - לבחור אסטרטגיה אחת (או meta או JS) כדי להימנע מכפילות ו-debug מיותר

3) Console warnings
   - חומרה: נמוכה
   - מיקום: pages/home/index.html כולל console.warn ב-catch של init
   - הערה:
     - זה כנראה מכוון (לוג דיבוג), אבל אם רוצים “אפס אזהרות” בפרודקשן אפשר להחליש/להוריד
